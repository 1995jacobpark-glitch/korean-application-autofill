# HWP 이력서 자동작성 웹서비스 기획서

작성일: 2026-04-19

## 1. 서비스 개요

공공기관, 지자체, 공기업 등에서 제공하는 서로 다른 HWP 이력서/위원 등록 신청서 양식에 사용자의 기본 인적사항, 자격증, 학력, 경력, 수상, 프로젝트 이력을 자동으로 배치해 주는 웹서비스를 만든다.

핵심은 “한 번 입력한 내 정보”를 여러 관공서 HWP 양식에 재사용하되, 최종 저장 전 사용자가 중간값을 직접 확인하고 수정할 수 있게 하는 것이다. 입력 파일과 출력 파일은 모두 HWP를 지원해야 한다.

## 2. 해결하려는 문제

현재 문제는 다음과 같다.

- 관공서마다 이력서와 평가위원 등록 신청서 양식이 다르다.
- 반복 입력 항목은 대부분 비슷하다. 예: 성명, 생년월일, 주소, 휴대폰, 이메일, 직장, 직위, 학력, 자격증, 경력, 주요 프로젝트.
- HWP 표 구조가 기관마다 달라 수작업 복사/붙여넣기가 자주 필요하다.
- 공고문 안에 신청서, 보안서약서, 개인정보 동의서가 함께 들어 있는 경우가 많아 작성 대상 서식을 사람이 찾아야 한다.
- AI가 자동으로 넣더라도 틀린 칸에 들어가면 위험하므로, 최종 HWP 생성 전 반드시 검수 가능한 중간 단계가 필요하다.

## 3. 제품 목표

1. 사용자가 기존 HWP 이력서를 업로드하면 개인 프로필 초안을 자동 추출한다.
2. 사용자가 관공서 HWP 공고문 또는 신청서 양식을 업로드하면 작성해야 할 서식과 빈칸을 자동 탐지한다.
3. AI가 각 빈칸에 들어갈 후보 값을 추천한다.
4. 사용자는 “중간값 편집 화면”에서 AI 추천값을 수정, 삭제, 확정할 수 있다.
5. 확정된 값으로 원본 양식의 표, 글꼴, 페이지 구성을 최대한 유지한 HWP 파일을 생성한다.
6. 제출서류 체크리스트와 누락 항목을 함께 보여준다.

## 4. 비목표

초기 버전에서는 다음을 하지 않는다.

- 모든 기관 양식을 100% 무검수 자동 제출 처리하지 않는다.
- 사용자의 동의 없이 개인정보 동의서의 체크박스를 자동 확정하지 않는다.
- PDF, DOCX, 이미지 스캔본 OCR은 초기 필수 범위에서 제외한다.
- 공공기관 이메일 발송까지 자동화하지 않는다. 제출 메일 주소와 제출 마감일 안내까지만 제공한다.

## 5. 샘플 문서 분석 결과

### 5.1 기존 이력서 HWP에서 추출 가능한 개인 프로필

사용자가 제공한 기존 이력서에는 다음 그룹이 들어 있다.

- 기본사항: 성명, 영문명, 이메일, 휴대폰, 주소
- 학력사항: 충북대학교, 충북대학교 대학원, 서울대학교 경영대학원 등
- 자격증: 수자원개발기술사, 토목기사1급, 방재전문인력인증서
- 개인능력: 어학, 컴퓨터 활용, 수상실적
- 취미/특기
- 주요 경력: 한국도로공사, 한국수자원공사, 주)이산 등
- 기타 경력: 겸임교수, 국토교통부 위원, 행정안전부 재해영향평가 심의위원 등
- 주요 참여 프로젝트: 해외사업, 국책사업, 수자원/수도 분야 사업

단, 샘플 이력서 기준으로 생년월일, 자택 전화번호, 직장 주소, 직장 전화번호, 팩스번호 등은 비어 있거나 별도 입력이 필요하다.

### 5.2 공고문 HWP에서 추출 가능한 작성 대상

제공된 공고문에는 다음이 포함되어 있다.

- 공고명: 공법선정위원회 평가위원(후보자) 공개모집 공고
- 모집기관: 정선군
- 대상사업: 역전 자연재해위험개선지구 정비사업
- 모집분야: 수자원, 상하수도, 기계
- 접수기간: 2026. 2. 6. ~ 2026. 2. 12.
- 접수방법: 전자메일 접수
- 제출서류:
  - 제안서 평가위원(후보자) 등록 신청서
  - 자격 증빙서류
  - 보안서약서
  - 개인정보 수집 및 이용 동의서

붙임 1 신청서에는 성명, 생년월일, 모집분야, 자택 주소, 휴대폰, 직장명, 직위, 직장 주소, 이메일, 학력, 경력, 자격증, 저서 및 논문, 기타사항이 필요하다.

붙임 2 보안서약서에는 소속, 성명, 날짜, 서명란이 필요하다.

붙임 3 개인정보 동의서에는 성명, 생년월일, 동의 여부, 날짜, 성명이 필요하다.

## 6. 핵심 사용자 흐름

### 6.1 최초 1회: 내 프로필 만들기

1. 사용자가 기존 이력서 HWP를 업로드한다.
2. 시스템이 HWP 본문과 표를 분석한다.
3. AI가 개인 프로필 초안을 만든다.
4. 사용자는 항목별로 확인하고 수정한다.
5. 저장된 프로필은 이후 모든 관공서 양식에 재사용된다.

### 6.2 매번 지원할 때: 관공서 양식 작성

1. 사용자가 공고문 HWP 또는 신청서 HWP를 업로드한다.
2. 시스템이 문서 안에서 작성 대상 서식을 찾는다.
3. AI가 빈칸/표/체크박스를 분석하고 프로필 필드와 매칭한다.
4. 사용자는 중간값 편집 화면에서 추천값을 검토한다.
5. 누락값, 낮은 신뢰도, 법적 확인 항목을 확인한다.
6. 최종 생성 버튼을 누르면 HWP 파일이 만들어진다.

### 6.3 반복 사용: 기관별 양식 학습

1. 사용자가 매핑을 수정하면 시스템이 그 기관/양식의 규칙을 저장한다.
2. 다음에 비슷한 양식을 올리면 저장된 규칙을 우선 적용한다.
3. AI는 새로운 칸이나 애매한 칸만 다시 추천한다.

## 7. 주요 화면 설계

### 7.1 홈 대시보드

표시 항목:

- 최근 작성 문서
- 내 프로필 완성도
- 누락된 필수 정보
- 저장된 기관별 템플릿
- 새 HWP 작성 버튼

주요 버튼:

- 기존 이력서로 프로필 만들기
- 관공서 양식 업로드
- 내 프로필 수정
- 작성 이력 보기

### 7.2 내 프로필 화면

섹션:

- 기본정보: 성명, 영문명, 생년월일, 휴대폰, 이메일, 주소
- 직장정보: 회사명, 직위, 직급, 부서, 주소, 전화, 팩스
- 학력
- 자격증
- 경력
- 위원/자문/심의 이력
- 수상
- 주요 프로젝트
- 저서 및 논문
- 서명/도장 이미지: 선택 기능

반복 항목은 표 형태로 관리한다. 예를 들어 자격증은 자격증명, 취득일, 번호, 발급기관, 비고를 가진다.

### 7.3 HWP 양식 업로드 화면

업로드 후 자동 표시:

- 문서 제목
- 포함된 서식 목록
- 제출서류 목록
- 제출 마감일
- 접수 이메일/담당자 연락처
- 작성해야 할 페이지 또는 붙임 번호

### 7.4 AI 매핑 검토 화면

왼쪽:

- 원본 HWP 미리보기
- 표/칸 단위 하이라이트
- 빈칸, 체크박스, 날짜, 서명란 표시

오른쪽:

- AI 추천값 목록
- 원본 프로필 값
- 적용될 값
- 신뢰도
- 수정 버튼
- “비워두기” 버튼
- “이 기관 양식 규칙으로 저장” 체크

예시:

| 양식 항목 | AI 추천값 | 신뢰도 | 사용자 액션 |
|---|---|---:|---|
| 성명 | 박노혁 | 높음 | 확정 |
| 모집분야 | 수자원 | 중간 | 사용자 확인 필요 |
| 생년월일 | 비어 있음 | 낮음 | 직접 입력 필요 |
| 자격증 보유현황 | 수자원개발기술사 외 2건 | 높음 | 행 선택/수정 |
| 개인정보 동의 | 동의함 | 법적 확인 | 사용자 직접 확인 |

### 7.5 중간값 편집 화면

이 화면이 제품의 핵심이다. AI가 바로 HWP를 만들지 않고, 먼저 `작성 초안 데이터`를 생성한다.

사용자는 다음을 수정할 수 있다.

- 칸별 텍스트
- 날짜 형식
- 몇 개의 경력/자격증을 넣을지
- 긴 경력을 축약할지 전체 입력할지
- 모집분야 선택
- 개인정보 동의 여부
- 서명/도장 포함 여부
- 빈칸으로 남길 항목

저장되는 중간값 예시:

```json
{
  "document_title": "정선군 평가위원 후보자 등록 신청서",
  "values": [
    {
      "target_id": "table_1.row_1.col_2",
      "label": "성명",
      "profile_field": "person.name_ko",
      "value": "박노혁",
      "confidence": 0.98,
      "status": "confirmed"
    },
    {
      "target_id": "table_1.row_1.col_9",
      "label": "모집분야",
      "profile_field": "application.recruit_field",
      "value": "수자원",
      "confidence": 0.74,
      "status": "needs_review"
    }
  ]
}
```

### 7.6 최종 확인 화면

표시 항목:

- 작성 완료율
- 필수값 누락 목록
- 낮은 신뢰도 항목 목록
- 법적 확인 필요 항목
- 제출서류 체크리스트
- HWP 미리보기

사용자가 최종 확인을 누른 뒤에만 HWP를 생성한다.

## 9. AI 매핑 전략

AI는 문서 전체를 자유롭게 수정하는 역할이 아니라, “어느 칸에 어떤 프로필 필드를 넣을지 추천하는 역할”만 맡긴다. 실제 HWP 편집은 규칙 기반 렌더러가 수행한다.

### 9.1 1단계: 문서 구조 분석

시스템이 HWP에서 다음을 추출한다.

- 문단 텍스트
- 표 구조
- 셀 텍스트
- 빈 셀 위치
- 체크박스 문자
- 날짜/서명 문구
- 붙임 번호와 제목

### 9.2 2단계: 후보 필드 생성

AI가 다음을 판단한다.

- “성명” 옆 빈칸은 `person.name_ko`
- “휴대폰” 옆 빈칸은 `person.mobile`
- “자격증 보유현황” 표에는 `licenses[]`
- “경력사항” 표에는 `careers[]`
- “서약자: 소속 성명”에는 `work.company`, `person.name_ko`
- “개인정보 수집·이용 동의”는 사용자 확인 필요

### 9.3 3단계: 신뢰도와 설명 제공

모든 추천값에는 신뢰도를 붙인다.

- 0.90 이상: 자동 적용 가능, 그래도 사용자가 최종 검토
- 0.70 ~ 0.89: 노란색 표시, 사용자 확인 필요
- 0.70 미만: 자동 적용 금지, 직접 입력 필요

### 9.4 개인정보 보호형 AI 호출

권장 설계:

- AI에는 원본 HWP의 라벨과 표 구조, 프로필 필드 이름만 보낸다.
- 실제 주민성 개인정보 값은 가능하면 로컬에서만 결합한다.
- 예: AI는 “성명 칸에는 `person.name_ko`를 넣어라”까지만 추천하고, 실제 “박노혁” 값 삽입은 서버/로컬 렌더러가 수행한다.
- 사용자가 허용한 경우에만 실제 값을 AI에 전달한다.

## 10. HWP 처리 구조

### 10.1 입력 HWP 읽기

필요 기능:

- HWP 문서의 텍스트 추출
- 표 구조 추출
- 섹션/페이지/붙임 구분
- 서식 제목 탐지
- 문서 버전 및 암호화 여부 확인

샘플 문서는 HWP 5.x 형식이며 암호화되어 있지 않아 파싱 가능했다.

### 10.2 출력 HWP 생성

출력도 반드시 HWP여야 하므로, 초기 제품에서는 Windows 환경의 HWP 자동제어 워커를 권장한다.

권장 방식:

1. 원본 HWP를 그대로 연다.
2. 셀/문단 위치를 찾아 확정된 값을 삽입한다.
3. 표 높이, 줄바꿈, 글자 크기, 페이지 넘김을 확인한다.
4. HWP로 다시 저장한다.

대안 방식:

- HWP를 HWPX로 변환한 뒤 XML을 조작하고 다시 HWP로 변환한다.
- 이 방식은 서버 자동화에는 유리할 수 있으나, 최종 HWP 변환 품질 검증이 필요하다.

### 10.3 미리보기

미리보기는 다음 중 하나로 제공한다.

- HTML 미리보기: 빠르지만 실제 HWP 줄바꿈과 다를 수 있음
- 이미지/PDF 미리보기: 실제 제출본 검수에 유리
- HWP 내보내기 후 다시 렌더링한 최종 미리보기: 가장 안전

MVP에서는 “HTML 빠른 미리보기 + 최종 HWP 생성 후 페이지 이미지 검수” 조합을 권장한다.

## 11. 시스템 아키텍처

### 11.1 권장 형태: 로컬 웹앱

개인정보와 HWP 자동제어를 고려하면 초기 버전은 사용자의 Windows PC에서 실행되는 로컬 웹앱이 적합하다.

구성:

- Frontend: React 또는 Next.js
- Backend API: FastAPI 또는 Node.js
- HWP Worker: Windows + 한글 프로그램 자동제어
- DB: SQLite 또는 PostgreSQL
- File Storage: 로컬 암호화 폴더
- AI Mapping Service: 외부 API 또는 로컬 모델 선택

사용자는 브라우저에서 `localhost`로 접속하지만, 파일은 PC 안에서 처리된다.

### 11.2 서버형 서비스로 확장할 경우

서버형은 여러 사용자가 쓸 수 있지만 다음 조건이 필요하다.

- HWP 처리용 Windows 서버
- 한글 프로그램 라이선스 검토
- 개인정보 암호화 저장
- 기관 제출서류 자동 삭제 정책
- 작업 파일 접근권한 분리
- 감사 로그

## 13. 샘플 공고문 적용 예시

정선군 공고문을 업로드했을 때 예상 흐름:

1. 시스템이 “붙임 1 등록 신청서”, “붙임 2 보안서약서”, “붙임 3 개인정보 동의서”를 작성 대상으로 탐지한다.
2. 제출서류 체크리스트에 자격증, 학위증명서, 재직증명서 등이 표시된다.
3. 모집분야 후보로 “수자원”을 추천한다. 이유는 프로필에 수자원개발기술사와 수자원/수도 분야 경력이 많기 때문이다.
4. 생년월일은 기존 이력서에 없으므로 직접 입력이 필요하다고 표시한다.
5. 신청서 학력에는 박사학위와 관련 학력 3~4개를 표 행에 배치한다.
6. 자격증에는 수자원개발기술사, 토목기사1급, 방재전문인력인증서를 배치한다.
7. 경력사항에는 한국수자원공사, 주)이산, 주요 심의위원 경력 중 사용자가 선택한 항목만 넣는다.
8. 보안서약서에는 소속과 성명, 날짜만 자동 입력한다.
9. 개인정보 동의서는 성명/생년월일을 입력하되, 동의 체크는 사용자가 직접 확정한다.

## 16. 개발 단계

### 16.1 1단계: 기술 검증

목표:

- 샘플 HWP를 읽고 표 구조를 추출한다.
- 샘플 신청서에 몇 개 필드를 삽입해 HWP로 저장한다.
- 원본과 출력본의 레이아웃 보존 수준을 확인한다.

완료 기준:

- 성명, 휴대폰, 이메일, 주소, 자격증 1개 이상이 HWP에 삽입된다.
- 출력 HWP가 한글 프로그램에서 정상적으로 열린다.

### 16.2 2단계: 프로필 DB 구축

목표:

- 기존 이력서에서 개인 프로필을 추출한다.
- 사용자가 추출값을 수정해 저장한다.

완료 기준:

- 기본정보, 학력, 자격증, 경력 테이블을 저장하고 다시 불러올 수 있다.

### 16.3 3단계: AI 매핑 및 중간값 편집

목표:

- 템플릿의 빈칸과 프로필 필드를 자동 연결한다.
- 사용자가 매핑값을 수정할 수 있다.

완료 기준:

- 모든 추천값에 신뢰도와 수정 상태가 표시된다.
- 낮은 신뢰도 항목은 최종 생성 전에 경고된다.

### 16.4 4단계: HWP 출력 안정화

목표:

- 원본 양식 유지율을 높인다.
- 표 글자 넘침, 행 높이, 줄바꿈 문제를 줄인다.

완료 기준:

- 샘플 5개 이상의 다른 기관 HWP 양식에서 제출 가능한 출력물이 생성된다.

### 16.5 5단계: 반복 사용 편의 기능

목표:

- 기관별 매핑 규칙 저장
- 제출서류 체크리스트
- 작성 이력 관리
- 프로필 버전 관리

## 17. 성공 기준

정량 기준:

- 기본 인적사항 매핑 정확도 95% 이상
- 자격증/학력/경력 표 매핑 정확도 85% 이상
- 한 문서 작성 시간 30분 이상에서 5분 이하로 단축
- 최종 HWP 열기 성공률 99% 이상

정성 기준:

- 사용자가 AI 추천을 신뢰하되, 최종 통제권은 사용자에게 있다고 느낀다.
- 관공서 양식이 바뀌어도 새 템플릿을 쉽게 적용할 수 있다.
- 개인정보가 외부로 나간다는 불안이 낮다.

## 20. 기획 결론

이 서비스의 핵심은 AI 자동작성 자체가 아니라, “원본 HWP 양식을 보존하면서 AI가 추천한 값을 사용자가 검수한 뒤 HWP로 다시 내보내는 구조”다.

따라서 제품은 다음 순서로 설계해야 한다.

1. 기존 이력서에서 내 프로필을 정확히 추출한다.
2. 관공서 HWP 양식에서 작성해야 할 칸을 정확히 찾는다.
3. AI는 칸과 프로필 필드를 추천한다.
4. 사용자는 중간값을 직접 수정한다.
5. 확정된 값만 HWP 렌더러가 원본 양식에 삽입한다.

이 방향이면 기관마다 다른 이력서 템플릿을 계속 새로 작성하는 부담을 크게 줄이면서도, 공공기관 제출 문서에 필요한 정확성과 검수 가능성을 확보할 수 있다.
